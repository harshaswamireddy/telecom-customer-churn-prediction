# Telecom Customer Churn Prediction

## Objective
Predict customer churn and derive actionable strategies to retain users in a highly competitive telecom environment.

## Dataset
`telecom_customer_data.csv` - A comprehensive dataset containing historical usage data, demographics, call duration, customer support complaints, and recharge frequency.

## Tools Used
* Python (Pandas, NumPy)
* Scikit-Learn
* ELI5 / SHAP
* SQL
* Microsoft PowerPoint

## Project Features
* Data Aggregation & Preprocessing of raw transactional logs.
* Binary Classification Machine Learning Model to forecast cancellations.
* Model Explainability to ensure transparent and interpretable predictions.
* Dynamic Customer Segmentation (e.g., At Risk, Loyal, Dormant).

## Key Insights
* A high volume of customer support tickets strongly correlates with an increased likelihood of churning.
* Low recharge frequency over consecutive months serves as a primary early warning indicator.
* The predictive model accurately flags 'At Risk' customers *before* they cancel their service.
* Explainability tools (like ELI5) highlight exactly why specific accounts are flagged, allowing for personalized, targeted interventions.

## Outcome
Successfully built a robust predictive framework that transitions customer service from reactive to proactive. The project empowers retention teams to focus their efforts on saving at-risk customers, thereby protecting recurring business revenue.
